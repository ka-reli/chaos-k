# Chaos-FX (SillyTavern FX-расширение)

FX-расширение, оформляющее текст постов богатыми HTML/CSS-эффектами через лёгкие
метки. Полная спецификация — в [`CLAUDE.md`](./CLAUDE.md).

## Статус

Закрыт первый вертикальный срез **«метка → экран»** (пункты 3–4 роадмапа):

- `data/registry.js` — единый источник правды: 16 настроений, 32 эффекта, 24 цвета, 14 форм.
- `src/parser.js` — парсер меток со стеком (токенайзер → AST → HTML), толерантность
  и предохранители (бюджет, потолок вложенности, `maxChars`, reduced-motion).
- `src/rotation.js` — ротация палитры: фильтр по накалу и настроениям, взвешенный
  рандом (приоритет мульти-настроенческим) → горстка эффектов/цветов + блок-подсказка
  для модели (`setExtensionPrompt`); опционально подмешивает инструкцию формы.
- `css/chaos-fx.css` — двойная палитра (тёмная/светлая) + контрастная обводка +
  все 32 эффекта (со статическими reduced-motion-вариантами).
- `demo/index.html` — standalone-стенд (открывается двойным кликом, без сервера):
  путь «метка → экран» + панель ротации палитры.
- `manifest.json` + `index.js` — обвязка SillyTavern: рендер меток в DOM сообщения,
  инъекция палитры (`setExtensionPrompt`), перехватчик контекста, кнопка
  «сделать читаемым», панель настроек.
- `test/run.js`, `test/run-rotation.js` — дымовые тесты (`node test/run.js`).

## Установка в SillyTavern

Расширение — это сам репозиторий (manifest в корне). Поставить можно двумя путями:

1. **Через UI:** Extensions → Install extension → вставить URL репозитория.
2. **Вручную:** склонировать в
   `SillyTavern/public/scripts/extensions/third-party/chaos-k/`, перезапустить ST.

После установки — панель **Chaos-FX** в настройках расширений: вкл/выкл, потолок
накала, активные настроения (ручной режим), бюджет, тема, reduced-motion. Режиссёр
(автоподбор настроений) появится позже — пока настроения задаются вручную.

## Синтаксис меток (зафиксирован)

| Ось         | Метка                          | Поведение |
|-------------|--------------------------------|-----------|
| Эффект      | `[fx:ember]…[/fx]`             | имя эффекта — параметр; известный применяется, неизвестный выкусывается без следа |
| Настроение  | `[ominous]…[/ominous]`        | разворачивается в случайный эффект из пула настроения (взвешенно по `weight`) |
| Цвет        | `[void]…[/void]`              | отдельная ось, навешивается на любой эффект через вложенность |
| Вложенность | `[fx:ember][blood]…[/blood][/fx]` | стек: каждый слой — свой спан |
| Авто-закрытие | `[/]`                       | закрывает ближайшую открытую метку |

Регистр и пробелы не важны, алиасы сводятся к канону. Голое `[слово]`, не совпавшее
ни с настроением, ни с цветом, считается обычным текстом — markdown-ссылки
`[текст](url)` и OOC-скобки не ломаются. Сырые метки игроку не протекают.

## Запуск

```bash
node test/run.js              # тесты парсера
node test/run-rotation.js     # тесты ротации
# demo/index.html — открыть в браузере
```
